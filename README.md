> **Graduation Project** | AI-powered dermatology assistant supporting English & Arabic  
> Built with RAG (Retrieval-Augmented Generation), FAISS, and LLaMA 3.3 via Groq

---

## рҹ“Ң Overview

DermaCam is an intelligent chatbot that answers questions about **8 common skin diseases** in both **English and Arabic**. It uses a Retrieval-Augmented Generation (RAG) pipeline вҖ” combining a local FAISS vector database with the LLaMA 3.3 70B model (served via Groq API) вҖ” to give accurate, context-grounded responses.

The system is specialized: it only answers questions within its knowledge base and clearly refuses off-topic queries, making it safe and reliable for a medical awareness context.

### RAG Flow (per user message)
1. **Detect language** вҖ” `langdetect` decides English vs. Arabic
2. **Embed query** вҖ” `BAAI/bge-m3` model encodes the query into a vector
3. **Retrieve chunks** вҖ” FAISS cosine similarity search; top-5 results above threshold `0.40` are returned
4. **Deduplicate** вҖ” max 1 chunk per disease, top-3 kept
5. **Build prompt** вҖ” retrieved context + conversation history injected into system prompt
6. **LLM generation** вҖ” Groq streams LLaMA 3.3 70B response
7. **Confidence badge** вҖ” displayed based on top chunk score (рҹҹў вүҘ75%, рҹҹЎ вүҘ55%, рҹ”ҙ below)

## вҡҷпёҸ Setup & Installation

### Prerequisites
- Python 3.9+
- A [Groq API key](https://console.groq.com/) (free tier available)

### 1. Clone the repository
```bash
git clone https://github.com/fodaa711/Skin-disease-diagnosis-.git
cd Skin-disease-diagnosis-/chatbot2
```

### 2. Install dependencies
```bash
pip install -r requirements.txt
```

### 3. Configure environment variables
Create a `.env` file in the `chatbot2/` directory:
```env
GROQ_API_KEY=your_groq_api_key_here
```

### 4. Build the vector index (first time only)
> Skip this step if `faiss_index.bin` and `chunks.pkl` already exist.

```bash
python embedder.py
```
This downloads the `BAAI/bge-m3` embedding model (~570MB on first run) and builds the FAISS index from `knowledge_base.json`.

### 5. Run the chatbot
```bash
streamlit run app.py
```
Open your browser at `http://localhost:8501`

## рҹ”Ң API Reference (for Backend Integration)

The chatbot exposes two callable functions from `pipeline.py`:

### `chat(query, history=[])` вҖ” Non-streaming
```python
from pipeline import chat

result = chat("What are the symptoms of eczema?", history=[])

# Returns:
{
  "answer":   str,    # Full LLM response
  "lang":     str,    # "en" or "ar"
  "chunks":   list,   # Retrieved knowledge chunks with scores
  "score":    float,  # Top chunk similarity score (0.0вҖ“1.0)
  "fallback": bool    # True if no relevant chunks found
}
```

### `chat_stream(query, history=[])` вҖ” Streaming
```python
from pipeline import chat_stream

stream, meta = chat_stream("Щ…Ш§ ЩҮЩҠ ШЈШ№ШұШ§Ш¶ Ш§Щ„ШЈЩғШІЩҠЩ…Ш§Шҹ", history=[])

# meta contains: lang, chunks, score, fallback
# stream is a Groq streaming generator вҖ” iterate to get tokens:
for chunk in stream:
    delta = chunk.choices[0].delta.content
    if delta:
        print(delta, end="", flush=True)
```

### History format
Pass conversation history as a list of role/content dicts:
```python
history = [
    {"role": "user",      "content": "What is vitiligo?"},
    {"role": "assistant", "content": "Vitiligo is a condition where..."},
]
```
History is capped at **20 messages** (10 turns) automatically.

## рҹҢҗ Exposing as a REST API (for Backend Team)

The current chatbot runs as a Streamlit app. To connect it with a frontend UI or other services, wrap `pipeline.py` in a FastAPI server:

```python
# api.py
from fastapi import FastAPI
from pydantic import BaseModel
from pipeline import chat

app = FastAPI()

class ChatRequest(BaseModel):
    query: str
    history: list = []

@app.post("/chat")
def chat_endpoint(req: ChatRequest):
    return chat(req.query, req.history)
```

Run with:
```bash
uvicorn api.py:app --host 0.0.0.0 --port 8000
```

For streaming, use FastAPI's `StreamingResponse` with `chat_stream()`.

## рҹ“Ҡ Confidence Scoring

| Badge | Score Range | Meaning |
|-------|-------------|---------|
| рҹҹў High confidence | вүҘ 75% | Strong match in knowledge base |
| рҹҹЎ Medium confidence | 55вҖ“74% | Partial match |
| рҹ”ҙ Low confidence | < 55% | Weak match (answer may be limited) |
| *(no badge)* | fallback | No relevant chunks found; LLM used general knowledge |

Confidence badge is **suppressed** for greetings and refusal responses.

## рҹӣЎпёҸ Safety & Boundaries

The chatbot is designed with clear guardrails:

- **No treatment advice** вҖ” explicitly refused; user directed to a licensed dermatologist
- **No diagnosis** вҖ” awareness tool only, not a clinical decision tool
- **Out-of-scope diseases** вҖ” politely refused with the list of supported diseases
- **Rude/inappropriate input** вҖ” calmly redirected without repeating offensive content
- **No specific doctor/hospital recommendations** вҖ” only general guidance to seek a specialist
